# Índice de documentación

Este directorio reúne toda la documentación funcional y técnica del proyecto de gestión de turnos para el lubricentro. Cada carpeta sigue la numeración del plan de trabajo del equipo y contiene los entregables asociados a esa fase.

## Guía rápida por carpetas

- `00 - Documento de Visión y Alcance`: definición del problema, objetivos, actores clave y criterios de éxito del producto.
- `01 - Analisis Inicial`: relevamiento del contexto actual, flujo de negocio y mapa funcional utilizados para comprender la operación.
- `02 - Modelado de Dominio`: diagramas y artefactos de modelado conceptual que describen entidades, relaciones y reglas del dominio.
- `03 - Elaboración del Product Backlog`: priorización inicial de historias, diagramas de actividad/estado y entregables de la fase 3.
- `04 - Validacion y Trazabilidad`: matriz de trazabilidad requisitos-historias y documentación de validación realizada con los interesados.
- `06 - UX-UI`: wireframes y documentación de diseño de interfaces creados para las vistas principales del sistema.
- `07- API REST`: relevamiento y lineamientos generales para la definición de la API REST del proyecto.
- `08 - Matriz de Permisos`: detalle de roles, permisos y endpoints asociados a la autenticación/autorización (incluye JWT por módulo).
- `09- Sprint 3 Retrospectiva grupal`: conclusiones del equipo respecto al sprint, con fortalezas, oportunidades y acciones de mejora.
